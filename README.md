## =====   DESCRIPCIÓN DEL PROYECTO   ===== ##
Este proyecto consiste en el desarrollo de un sitio web de tipo e-commerce llamado Onixia, una marca creada por mí con el objetivo de representar una identidad visual propia orientada a un diseño moderno y minimalista.
La idea principal de Onixia no es solo simular una tienda online, sino también funcionar como una base para futuros proyectos personales, donde pueda aplicar y demostrar mis conocimientos en desarrollo y diseño web. De esta forma, el sitio también se plantea como parte de mi portfolio profesional.
Además, el diseño visual del sitio fue acompañado por la creación de un logo propio, desarrollado especialmente para la marca Onixia, reforzando así su identidad y coherencia visual.
El propósito general del sitio es integrar tanto el desarrollo técnico (HTML y CSS) como la construcción de una marca personal, combinando diseño, estructura y experiencia de usuario en un mismo proyecto.

## =====   ESTRUCTURA   ===== ##
Para realizar este proyecto trabajé con las bases de HTML y CSS, entendiendo cómo se arma una página web desde cero.
Primero utilicé la estructura principal de HTML, que es donde se organiza todo el contenido del sitio. Dentro de esa estructura se encuentra el <head>, donde van configuraciones generales de la página, y el <body>, que es donde se ve todo lo que aparece en la web.

Dentro del body fui organizando el contenido usando distintas etiquetas. Por ejemplo:
<h1> y <h2> para los títulos,
<p> para los textos,
<div> y <section> para agrupar y ordenar las distintas partes de la página.
Esto me permitió separar cada sección del sitio, como el inicio, la tienda y el contacto, de una forma clara.

Por otro lado, utilicé CSS para darle estilo a la página, como colores, tamaños, espacios y la forma en la que se acomodan los elementos.
También trabajé con clases para poder reutilizar estilos, especialmente en componentes como las tarjetas de productos, y lograr que todo tenga una misma estética.Además, usé Flexbox para ordenar los elementos en pantalla, lo que me ayudó a que las tarjetas se vean alineadas y más organizadas.


## =====   EXPERIENCIA   ===== ##
Para la realización de este proyecto conté con una base previa en desarrollo front-end, adquirida a partir de trabajos anteriores en la facultad, como el sitio web Santa Ramona, donde participé en la parte visual y de estructura. Además, tengo conocimientos básicos en temas de seguridad, lo que también influyó en la forma en la que fui pensando el desarrollo del sitio.

Por otro lado, mi interés y formación en diseño gráfico me ayudaron mucho durante todo el proceso, ya que pude pensar el sitio desde lo visual y no solo desde lo técnico. Esto hizo que el proyecto tenga una identidad más propia, pudiendo diseñarlo según mi estilo y las ideas que quería transmitir.

Al principio me costó bastante definir qué quería mostrar y cómo quería que se vea la página, pero con el tiempo fui ordenando mejor las ideas. Primero imaginé el concepto general y diseñé la identidad de la marca Onixia, incluyendo el logo. Antes de empezar a programar, pensé el diseño de forma más visual, incluso haciendo bocetos y tratando de anticipar cómo se vería todo una vez terminado.

Para inspirarme, utilicé mucho Pinterest, donde busqué referencias de sitios web, estilos, paletas de colores e incluso imágenes para los productos. Esto me ayudó bastante a definir la estética general y mantener una coherencia visual en todo el sitio. Una vez que tenía una idea más clara, empecé a llevar todo al código, probando cómo se veía cada parte y ajustando detalles a medida que avanzaba. Durante este proceso también utilicé herramientas como inteligencia artificial (ChatGPT y CLAUDE) como apoyo en algunas partes, especialmente en CSS, pero siempre tratando de entender el código y adaptarlo a mi diseño, en lugar de copiarlo directamente.

Otra herramienta que me ayudó muchísimo fue el inspeccionar, ya que me permitió ver cómo estaban hechos los elementos, probar cambios en tiempo real y ubicar rápidamente estilos como colores, tamaños o posiciones. Esto me sirvió mucho para entender mejor cómo funcionaba todo y poder corregir errores.

Muchos de los conocimientos utilizados provienen del curso, pero no los apliqué de forma literal, sino que los fui adaptando y modificando según lo que quería lograr. En general, el proyecto fue un proceso de prueba y error, donde fui aprendiendo, experimentando y ajustando detalles hasta llegar a un resultado final que representa tanto mis conocimientos como mi forma de diseñar.
